# φ FinHealer — System Flow v3.1

## הארכיטקטורה בשורה אחת
מוח AI אחד (PhiBrain) מנהל את כל התקשורת. הוא **orchestrator** — מפעיל handlers מתמחים, לא prompt אחד גדול. שני ערוצים (WhatsApp + Dashboard) כותבים וקוראים מאותו DB.

---

## עקרונות ליבה

### Data-Gating, לא Time-Gating
| שלב | תנאי כניסה |
|------|------------|
| ניתוח התנהגות | 2+ חודשי נתונים |
| יעדים | 2+ חודשים + ניתוח הושלם |
| תקציב | 3+ חודשי נתונים |
| ניטור | תקציב קיים |

### כלל השתיקה
- **מקסימום הודעה יזומה אחת ביום**
- הודעה קודמת לא נענתה → **לא שולחים עוד**
- "תפסיק" / "לא עכשיו" = **cooldown 48 שעות**
- **שבוע ראשון: אפס הודעות יזומות**
- שעת פנייה = **לפי engagement של המשתמש**

### Source of Truth אחד
תיקון בדשבורד = אותו `learnFromCorrection` כמו בוואטסאפ.

## שלב 7: ניטור שוטף

### מעקב הוצאות
```
"סופר 450" → סיווג + שמירה + עדכון תקציב + תגובה קצרה
"בטל" → מחיקת אחרונה
```

### PhiBrain — Orchestrator Architecture

```
PhiBrain(userId, event) → {

  // 1. טען קונטקסט מלא
  context = loadFullContext(userId)
  // פרופיל + DNA + תנועות + תקציב + יעדים + שיחות + timing

  // 2. בדוק כלל השתיקה (scheduled_check בלבד)
  if (event.type === 'scheduled_check' && silenceRule(context))
    → return { silent: true }

  // 3. זהה intent
  intent = classifyEvent(event, context)
  // expense_log | document_upload | question | correction |
  // greeting | summary_request | goal_request | scheduled_check

  // 4. בחר handler מתמחה
  handler = routeToHandler(intent)
  // handlers: ExpenseHandler, ClassifyHandler, SummaryHandler,
  //   GoalHandler, BudgetHandler, CoachingHandler, CorrectionHandler

  // 5. handler מבצע ומחזיר תגובה
  response = handler.execute(context, event)

  // 6. התאם לפרופיל אישי (טון, אורך, שפה)
  response = adjustToProfile(response, context.profile)

  // 7. עדכן פרופיל אם למדנו משהו חדש
  profileUpdates = inferFromInteraction(event, response)

  return { message, actions, profileUpdates }
}
```

**הנקודה הקריטית:** PhiBrain הוא **orchestrator**, לא monolith. ה-AI מחליט **מה** לעשות. ה-handlers יודעים **איך**.

## Error Handling & Fallbacks

```
Gemini timeout (10s)?
    → שכבות 0-2 בלבד, pending לשאר, retry בשקט בעוד שעה
    → המשתמש לא יודע

GreenAPI חסום?
    → תור (alerts table) → retry 5 דקות × 3 → alert לאדמין

OCR נכשל?
    → "לא הצלחתי לקרוא. אפשר לנסות פורמט אחר?"
    → שמור קובץ ל-retry

PhiBrain נכשל?
    → fallback ל-state machine הישנה
    → המשתמש תמיד מקבל תגובה
```

## φ Score — הגדרה

```
φ Score (0-100) = weighted average:

  30% — עמידה בתקציב
        avg(100 - |percentage_used - 100|) על פני כל הקטגוריות
        100 = בול בתקציב, 0 = חריגה כפולה

  25% — יחס חיסכון
        (הכנסות - הוצאות) / הכנסות × 100
        cap ב-25% (מעל = מושלם)

  20% — עקביות
        streak ימי דיווח / 30 × 100
        + bonus אם אין חריגות חוזרות באותה קטגוריה

  15% — התקדמות ליעדים
        avg(percentage) על פני כל היעדים הפעילים

  10% — מגמה
        השוואת חיסכון ממוצע 3 חודשים אחרונים vs 3 לפני
        שיפור = 100, יציב = 50, הידרדרות = 0
```

PhiBrain: "הציון עלה מ-68 ל-72 כי צמצמת מסעדות וה-streak שלך כבר 12 יום."
